# 功能沉淀 — 把已有代码录入团队功能池

> ⚠️ 注意：**不要给项目新增任何功能、不要修改任何代码。**
> 你的任务是：读取项目里**已经写好的代码**，识别其中可复用的 UI 功能模块，
> 列出清单让设计师选择，然后把选中的功能整理成卡片保存到团队功能池。

## 第一步：扫描项目，列出已有功能模块

当前项目中已有的源文件：

!`find . \( -name "*.vue" -o -name "*.jsx" -o -name "*.tsx" -o -name "*.html" -o -name "*.css" \) -not -path "*/node_modules/*" -not -path "*/.git/*" 2>/dev/null | head -60`

仔细读取上方**已存在的文件**，识别其中**已经实现**的独立 UI 功能模块（导航栏、轮播、表单、弹窗、动效等）。

**绝对不能虚构或推测未实现的功能，只列出代码里真实存在的。**

严格使用以下格式展示：

---

我扫描了项目中**已有的代码**，发现以下 **N 个**已实现、可录入功能池的功能，请告诉我要保存哪些（回复序号如 `1 3` 或 `全部`）：

- [ ] **① 功能名称**
  📁 `文件路径`
  💬 一句话说明（20字以内）

- [ ] **② 功能名称**
  📁 `文件路径`
  💬 一句话说明

---

## 第二步：逐个处理每个选中的功能

对每个选中的功能依次执行：

### 2a. 反推有效的需求描述

读取该功能**已有的代码文件**，分析其实现逻辑，生成「如果从零让 AI 复现这个功能，应该怎么描述」的草稿。
（这一步是总结已有代码，不是生成新代码）

描述必须包含：
- 视觉效果和交互行为（用户看到/操作了什么）
- 关键数值参数（尺寸、时间、断点等）
- 状态变化（hover / active / scroll / 移动端等）

### 2b. 向我追问补充信息

展示草稿后询问：

---

这是我根据代码反推的需求描述草稿：

「[草稿内容]」

请帮我补充以下信息（直接说「没有」可跳过）：
1. 这个描述有没有漏掉什么关键约束？
2. 有没有一开始忘说、后来才补充的描述？
3. 有没有已验证的变体版本？（如深色版、带动画版等）

---

### 2c. 生成功能卡片 SKILL.md

整合代码分析 + 我的补充，按 `~/feature-pool/TEMPLATE.md` 格式生成 SKILL.md。

分类参考：
- `navigation`：导航栏、侧边栏、面包屑
- `hero`：首屏 Banner、视差滚动
- `content`：卡片列表、时间轴、FAQ
- `forms`：联系表单、预约、登录框
- `ecommerce`：商品卡、购物车、价格表
- `media`：轮播、相册、视频播放器
- `interactive`：弹窗、抽屉、标签页、折叠面板
- `layout`：页脚、响应式容器、网格
- `animation`：滚动触发动画、入场动效
- `industry`：行业特定模板

### 2d. 生成 preview.html

基于功能代码生成一个纯 HTML/CSS/JS 的交互 demo 文件：
- 不依赖任何外部框架
- 展示完整交互行为（动画、响应式、状态变化）
- 占位内容使用通用文字
- 能独立在浏览器中运行

### 2e. 单个功能汇总确认

每处理完一个功能展示摘要：

---

**「功能名称」** 已生成 ✓

| 字段 | 内容 |
|------|------|
| 分类 | `category` |
| 标签 | `tag1, tag2` |
| 描述 | 一句话描述 |
| 需求描述预览 | （前50字…） |

有需要修改的请直接说，或继续处理下一个。

---

## 第三步：汇总确认后一键推送 PR

所有功能处理完毕后展示汇总，等待我回复「确认」后**立即自动执行**以下步骤：

!`cd ~/feature-pool && git checkout main && git pull origin main 2>&1 | tail -3`

```bash
cd ~/feature-pool

# 创建新分支
git checkout -b feat/功能英文名-$(date +%Y%m%d)

# 将生成的文件写入对应目录
# features/<category>/<feature-name>/SKILL.md
# features/<category>/<feature-name>/preview.html

# 更新 INDEX.md
git add .
git commit -m "feat: 新增功能卡片 — 功能中文名"
git push origin feat/功能英文名-$(date +%Y%m%d)

# 创建 PR
gh pr create \
  --title "新增功能卡片：功能中文名" \
  --body "**功能名称**：功能中文名\n**分类**：category\n**提交人**：$(git config user.name)"
```

推送完成后告知我：

---

✅ **PR 已提交，等待 PM 审核**

| | |
|---|---|
| PR 链接 | [GitHub PR URL] |
| 包含功能 | N 个 |

审核通过后，团队所有设计师执行 `git pull` 即可同步使用。
