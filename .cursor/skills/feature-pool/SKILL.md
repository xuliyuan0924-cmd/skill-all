# 功能检索与复用

## 第一步：检索匹配功能

读取 `~/feature-pool/INDEX.md`，根据设计师描述的需求关键词，匹配最相关的 1-3 个功能。

展示结果，格式如下：

```
在功能池中找到以下匹配结果：

① 响应式粘性导航栏
   效果：滚动吸顶，移动端折叠汉堡菜单，含背景色过渡动画
   分类：navigation · 复杂度：中
   预览：在浏览器打开 ~/feature-pool/features/navigation/sticky-header/preview.html

② 全屏侧边抽屉导航
   效果：点击图标后从左侧滑出全屏遮罩菜单
   分类：navigation · 复杂度：中
   预览：在浏览器打开 ~/feature-pool/features/navigation/drawer-nav/preview.html

输入序号选择（如「用①」），或输入「都不对」重新描述需求：
```

如果未找到匹配功能，回复：

```
功能池中暂无匹配的功能。
你可以：
1. 换个关键词重新搜索（如「导航」「菜单」「header」）
2. 直接描述需求，我来帮你实现，完成后可用 /feature-add 沉淀到池子
```

---

## 第二步：展示将要使用的需求描述

设计师选择后，读取对应功能的 SKILL.md，完整展示「有效的需求描述」和「关键约束」：

```
好的，将使用「响应式粘性导航栏」这个配方。

━━━━ 将使用以下需求描述 ━━━━

「帮我做一个粘性导航栏，要求：
- 桌面端水平布局，移动端折叠为汉堡菜单
- 滚动超过 80px 后背景从透明变为白色，带轻微阴影
- Logo 在左，导航链接在右
- 移动端点击汉堡图标，菜单从上向下展开
- 当前页链接高亮显示」

⚠️ 关键约束（已内置，无需重复说明）：
- 滚动触发距离 80px
- 移动端断点 768px
- 包含当前页 active 状态

━━━━━━━━━━━━━━━━━━━━━━━━

请告诉我当前项目需要补充的信息（可直接说「没有，开始做」）：
- 技术栈（Vue / React / 纯 HTML）
- 品牌色 / 字体
- 导航项名称和数量
- 其他特殊要求
```

---

## 第三步：整合执行

将「有效需求描述 + 关键约束 + 设计师补充」合并，按当前项目的技术栈生成完整代码。

生成完成后提示：

```
✅ 已完成「响应式粘性导航栏」

如果这次实现中你发现了新的约束或更好的描述方式，
可以用 /feature-add 更新功能池，让大家都受益。
```

---

## 注意事项

- 优先匹配标签、分类，其次匹配功能描述中的关键词
- 如果功能池为空或匹配度低，直接告知设计师，不要强行匹配
- 展示预览路径时，使用绝对路径方便设计师直接在 Finder 找到文件
- 执行时严格使用 SKILL.md 中的有效描述作为基础，不要自行改写核心逻辑
